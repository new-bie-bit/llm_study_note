# 📊 文本生成评估（高密度版）

---

# 一、统一认知框架（最重要）

> 🧠 所有评估本质：

```

评估 = similarity(reference, prediction)

```

---

## 不同方法的 similarity 定义：

| 方法 | similarity 本质 |
|------|----------------|
| ROUGE | 字符串重叠（symbolic） |
| BLEU | n-gram precision |
| BERTScore | embedding cosine（semantic） |
| LLM Judge | learned function（cognitive） |
| Accuracy | exact match |

---

# 二、ROUGE：结构/覆盖匹配（Symbolic）

## 核心思想

> 比较 **token 重叠 + 顺序结构**

---

## 方法

| 方法 | 特点 |
|------|------|
| ROUGE-1 | unigram（无序，bag-of-words） |
| ROUGE-2 | bigram（局部连续） |
| ROUGE-L | LCS（全局顺序） |

---

## ROUGE-L（核心）

### 定义

> LCS = 最长公共子序列（顺序一致，不要求连续）

---

### 示例

```

A: 蛋白质 结构 预测 方法
B: 深度学习 蛋白质 预测 方法

```
```

LCS = 蛋白质 → 预测 → 方法
LCS = 3

```

---

## 计算

### DP定义

\[
dp[i][j] = LCS(A[:i], B[:j])
\]

### 转移

\[
\text{same: } dp[i][j] = dp[i-1][j-1] + 1
\]

\[
\text{diff: } dp[i][j] = \max(dp[i-1][j], dp[i][j-1])
\]

---

## 评分

\[
R = \frac{LCS}{m}, \quad P = \frac{LCS}{n}, \quad F1 = \frac{2PR}{P+R}
\]

---

## 优缺点

✔ 保留顺序（结构敏感）  
✔ 适合摘要/标题  

❌ 不理解语义  
❌ 同义词/改写失败  

---

## 本质

```

离散空间 + 硬对齐（exact match）

```

---

# 三、BERTScore：语义匹配（Semantic）

---

## 为什么出现（承接 ROUGE）

> ❗ ROUGE 问题：

```

结构 ≠ 折叠 → 判错（但语义相同）

```

👉 **BERTScore = 解决“语义等价但字符串不同”问题**

---

## 核心思想

> embedding 空间 + soft alignment

---

## 输入

```

Reference: x₁...x_m
Prediction: y₁...y_n

```

---

## 计算步骤

### 1️⃣ embedding（BERT）

\[
x_i, y_j ∈ \mathbb{R}^d
\]

---

### 2️⃣ 相似度矩阵

\[
S_{ij} = \cos(x_i, y_j)
\]

---

### 3️⃣ soft alignment（核心）

#### Recall

\[
R = \frac{1}{m} \sum_i \max_j S_{ij}
\]

#### Precision

\[
P = \frac{1}{n} \sum_j \max_i S_{ij}
\]

---

### 4️⃣ F1

\[
F1 = \frac{2PR}{P+R}
\]

---

## 本质

```

连续空间 + 软对齐（soft matching）
≈ frozen attention matching

```

---

## 优缺点

✔ 理解语义  
✔ 支持同义词 / 改写  

❌ 不考虑顺序  
❌ 依赖 embedding  
❌ greedy 非全局最优  

---

# 四、LLM-as-a-Judge：认知评估（Cognitive）

---

## 为什么出现（承接 BERTScore）

> ❗ BERTScore 仍然问题：

```

语义相似 ≠ 正确 / 高质量

```

👉 需要评估：

- 是否正确（fact）
- 是否完整
- 是否符合学术规范

---

## 核心思想

> 用大模型作为 evaluator：

\[
Score = f_\theta(input, output, reference)
\]

---

## 输入

```

任务描述 + reference + prediction

```

---

## 输出

```

Score + Reason

```

---

## 评估维度（可定制）

- Fluency（流畅性）
- Relevance（相关性）
- Faithfulness（真实性）
- Completeness（完整性）
- Academic style（⭐可扩展）

---

## 常见形式

### 1️⃣ 打分

```

Score: 8.5 / 10

```

---

### 2️⃣ Pairwise（更稳定）

```

A vs B 哪个更好？

```

---

## 本质

```

learned evaluation function
≈ human preference modeling

```

---

## 优缺点

✔ 最接近人类判断  
✔ 可多维评估  
✔ 可适配领域（科研）  

❌ 不稳定（prompt敏感）  
❌ 有bias  
❌ 成本高  

---

# 五、三种方法的演化（关键主线）

---

## 🔥 核心逻辑

```

ROUGE → BERTScore → LLM Judge

```

---

## 对应能力提升

| 层级 | 方法 | 能力 |
|------|------|------|
| 表层 | ROUGE | 字符串匹配 |
| 语义 | BERTScore | 语义对齐 |
| 认知 | LLM Judge | 质量判断 |

---

## 一句话串起来（建议记住）

> ROUGE 解决“是否匹配”，BERTScore 解决“是否语义一致”，LLM-as-a-Judge 解决“是否高质量”，三者构成从符号层 → 语义层 → 认知层的评估演化路径。

---

# 六、最终统一理解（压缩版）

```

ROUGE      = exact match（结构）
BERTScore  = semantic match（语义）
LLM Judge  = quality judgment（认知）

```

---

# 七、项目评估体系（可用于论文/简历）

```

1. ROUGE-L → 结构对齐
2. BERTScore → 语义一致性（可扩展）
3. Accuracy → 分类正确性
4. LLM Judge → 学术质量评估（future work）

```
